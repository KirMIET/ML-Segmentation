## 🏗️ Архитектура

Пайплайн использует **ансамбль из двух архитектур** нейронных сетей:

| Модель | Тип | Энкодер | Описание |
|--------|-----|---------|----------|
| **SegFormer** | Transformer (ViT-based) | MiT-B4 | Современная архитектура на основе трансформеров для сегментации |
| **U-Net** | CNN | ConvNeXt-Base | Классическая U-Net++ с современным энкодером ConvNeXt |

Обе модели обучаются с использованием **K-fold кросс-валидации** (4 фолда), что позволяет получить более надежные предсказания.

---

## ✨ Ключевые особенности

### Обучение
- **K-fold кросс-валидация** — автоматическое разделение на 4 фолда
- **EMA (Exponential Moving Average)** — скользящее среднее весов модели (decay=0.999) для улучшения обобщения
- **AMP (Automatic Mixed Precision)** — смешанная точность для ускорения обучения и экономии памяти
- **Градиентный клиппинг** — предотвращение взрыва градиентов (max_norm=1.0)
- **Ранняя остановка** — по метрике val_dice с настраиваемым patience
- **CosineAnnealingWarmRestarts** — косинусный планировщик скорости обучения с рестартами

### Функция потерь
Комбинированная функция потерь:
```
Loss = 0.5 × Lovász-Softmax + 0.5 × BCE
```
- **Lovász-Softmax** — оптимизация метрики IoU напрямую
- **BCE (Binary Cross-Entropy)** — стабильная оптимизация вероятностей

### Аугментации данных
Расширенный пайплайн аугментаций для повышения робастности модели:
- Горизонтальное и вертикальное отражение
- Случайные повороты на 90°
- Аффинные преобразования (масштабирование, поворот, смещение)
- Изменение яркости и контраста
- Изменение тона и насыщенности
- Гауссовский шум и размытие
- Нормализация по статистике ImageNet

## 🚀 Обучение модели

### Запуск обучения

```bash
python train_model_v4.py
```

### Что происходит при обучении

Для каждого фолда (1-4) и каждой модели (SegFormer, U-Net):
1. Создаются трансформы аугментации с нормализацией под конкретный энкодер
2. Инициализируется модель с предобученными весами ImageNet
3. Запускается цикл обучения с:
   - AMP для экономии памяти
   - EMA для стабилизации весов
   - Градиентным клиппингом
   - CosineAnnealingWarmRestarts scheduler
4. Валидация проводится для обеих версий: обычной и EMA
5. Сохраняются лучшие чекпоинты и история обучения

### Основные гиперпараметры

| Параметр | Значение |
|----------|----------|
| Размер изображения | 352×352 |
| Batch size (SegFormer) | 8 |
| Batch size (U-Net) | 8 |
| Learning rate (SegFormer) | 1e-4 |
| Learning rate (U-Net) | 5e-4 |
| Weight decay (SegFormer) | 5e-2 |
| Weight decay (U-Net) | 3e-2 |
| Макс. эпох | 50 |
| Early stopping patience | 10-15 (зависит от модели) |
| EMA decay | 0.999 |
| Порог бинаризации | 0.5 |

## 🔍 Инференс

### Запуск

```bash
python inference.py
```

### Возможности

- **Ансамблирование**: взвешенное усреднение предсказаний от 7 моделей (4 SegFormer + 3 U-Net)
- **TTA (Test-Time Augmentation)**: для каждой модели применяется 6 аугментаций:
  - Оригинальное изображение
  - Горизонтальное отражение
  - Вертикальное отражение
  - Повороты на 90°, 180°, 270°
- **Автоматическая загрузка чекпоинтов**: извлекает конфигурацию модели из сохраненного чекпоинта
- **AMP инференс**: ускорение предсказаний с mixed precision

### Выходные данные

Создается CSV-файл `submissions/submission_simple.csv` с колонками:
- `ImageId` — имя файла изображения
- `mask` — JSON-сериализованная бинарная маска (H×W)

### Настройка

В файле `inference.py` укажите:
```python
TEST_IMAGES_DIR = Path(r"dataset/best_dataset/test_images")  # Путь к тестовым изображениям
CHECKPOINT_PATHS = [...]  # Список путей к чекпоинтам
THRESHOLD = 0.47  # Порог бинаризации
```

## ⚖️ Оптимизация весов ансамбля

### Запуск

```bash
python src/ensemble_weights.py
```

### Описание

Скрипт автоматически:
1. Находит файлы истории обучения для каждого чекпоинта
2. Извлекает лучшую метрику `val_dice` (для EMA или обычной версии)
3. Вычисляет веса моделей методом **softmax нормализации**:
   ```
   weight_i = exp(val_dice_i × temperature) / Σ(exp(val_dice_j × temperature))
   ```
4. Сохраняет веса в JSON-файл

### Конфигурация

```python
CHECKPOINT_PATHS = [...]  # Пути к чекпоинтам
WEIGHT_METHOD = "softmax"  # или "linear"
SOFTMAX_TEMPERATURE = 20.0  # Чем выше, тем больше разница в весах
OUTPUT_WEIGHTS_PATH = "submissions/ensemble_weights.json"
```

## 🎯 Поиск оптимального порога

### Запуск

```bash
python src/find_optimal_threshold.py
```

### Описание

Скрипт ищет оптимальный порог бинаризации для ансамбля:
1. Запускает инференс на валидационной выборке
2. Вычисляет Dice score для диапазона порогов (0.10 - 0.70 с шагом 0.01)
3. Находит порог с максимальным средним Dice
4. Сохраняет результаты в JSON

### Конфигурация

```python
VAL_DIR = Path(r"dataset/dataset_fold/fold_2/val")  # Валидационная выборка
THRESHOLD_SEARCH_RANGE = (0.10, 0.70)  # Диапазон поиска
THRESHOLD_STEP = 0.01  # Шаг
```

## 📊 Выходные данные

### После обучения

В директории `model_checkpoints/` для каждой модели и фолда создаются:
- `best_{Model}_fold_{N}.pth` — лучшая модель по val_dice
- `ema_best_{Model}_fold_{N}.pth` — лучшая EMA-модель
- `last_{Model}_fold_{N}.pth` — последняя модель
- `history_{Model}_fold_{N}.csv` — история обучения (метрики по эпохам)

### Структура чекпоинта

```python
{
    "fold": int,                    # Номер фолда
    "epoch": int,                   # Номер эпохи
    "model_state_dict": dict,       # Веса модели
    "ema_state_dict": dict,         # Веса EMA
    "optimizer_state_dict": dict,   # Состояние оптимизатора
    "val_dice": float,              # val_dice обычной модели
    "val_dice_ema": float,          # val_dice EMA-модели
    "config": {                     # Конфигурация модели
        "MODEL_NAME": str,
        "ENCODER_NAME": str,
        "ENCODER_WEIGHTS": str,
        "IMG_SIZE": int,
    }
}
```

## 📝 Метрики

Пайплайн отслеживает следующие метрики:
- **Dice Score** — основная метрика оптимизации
- **IoU** — дополнительная метрика
- **Loss** — значение функции потерь

Все метрики вычисляются как для обычной модели, так и для EMA-версии.
